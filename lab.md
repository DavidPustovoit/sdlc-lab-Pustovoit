Тема: Software Development Life Cycle (SDLC) — Життєвий цикл програмного забезпечення
Виконав: Пустовойт Давид
Варіант: 15 — Трекер фільмів і серіалів

1. Планування
Мета: Створити зручний додаток для зберігання списків фільмів та серіалів, який дозволить користувачеві не забувати, що він хоче подивитися, та вести історію вже переглянутого контенту.

2. Аналіз вимог (User Stories)
1. Як користувач, я хочу додавати фільм у список «Буду дивитися», щоб не загубити цікаву назву. ✅ (Must have)
2. Як користувач, я хочу переміщувати фільм у список «Переглянуто» та ставити йому оцінку (1–10). ✅ (Must have)
3. Як користувач, я хочу мати можливість шукати фільми у своїх списках за назвою.
4. Як користувач, я хочу додавати коментар до переглянутого фільму, щоб зберегти враження.
5. Як користувач, я хочу фільтрувати список за типом (фільм або серіал).

3. Дизайн (Прототип)


5. Реалізація (Псевдокод)
Опис алгоритму функції переміщення фільму до списку переглянутих:

function markAsWatched(movieID, userRating):
    // Знаходимо фільм у базі даних
    movie = database.find(movieID)

    if movie is empty:
        return "Помилка: Фільм не знайдено"

    // Змінюємо статус і зберігаємо оцінку
    movie.status = "WATCHED"
    
    if userRating < 1 OR userRating > 10:
        return "Помилка: Оцінка має бути від 1 до 10"
    
    movie.rating = userRating
    movie.watchedDate = currentDate()

    // Зберігаємо зміни
    database.update(movie)
    return "Фільм успішно перенесено у переглянуті"

5. Тестування
1. Тест 1: Користувач натискає "Позначити як переглянуте" на фільмі "Дюна" → фільм зникає зі списку "Планую" і з'являється в "Переглянуто".
2. Тест 2: Спроба додати фільм без назви → система показує повідомлення "Будь ласка, введіть назву".
3. Тест 3: Введення оцінки "12" (при шкалі до 10) → система не зберігає оцінку і підсвічує поле червоним.

6. Висновки
Для розробки цього трекера найкраще підходить модель Agile. Чому:
1. Гнучкість: Спочатку можна зробити прості списки (MVP). Пізніше, базуючись на відгуках, можна додати складніші функції, наприклад, завантаження постерів з інтернету, статистику за рік або соціальні функції (ділитися списками з друзями).
2. Швидкий результат: Користувач зможе користуватися базовою версією програми вже після першого спринта (ітерації).
3. Waterfall не підходить, бо у розважальних додатках вимоги часто змінюються в процесі створення інтерфейсу.
