**Варіант:** 15 — Трекер фільмів і серіалів

## 1. Планування
**Мета:** Створити мобільний застосунок для зручного ведення списків фільмів та серіалів («Хочу подивитися», «Переглянуто»), що дозволить користувачеві зберігати враження та планувати дозвілля.

## 2. Аналіз вимог (User Stories)
1. Як користувач, я хочу додавати фільм у список «Планую подивитися», щоб не забути його назву.
2. Як користувач, я хочу переміщувати фільм у список «Переглянуто» та ставити йому оцінку (від 1 до 10).
3. Як користувач, я хочу шукати фільми у своїй базі за назвою.
4. Як користувач, я хочу бачити загальну статистику часу, витраченого на перегляд серіалів.
5. Як користувач, я хочу залишати текстові нотатки до переглянутих фільмів.

## 4. Реалізація (Псевдокод)
**Функція: Додавання фільму до списку переглянутих**

```
function markMovieAsWatched(movieID, rating):
    // 1. Перевірка валідності оцінки
    if rating < 1 OR rating > 10:
        return "Помилка: Оцінка має бути від 1 до 10"

    // 2. Пошук фільму в базі даних
    movie = database.find(movieID)
    
    if movie is null:
        return "Помилка: Фільм не знайдено"

    // 3. Оновлення статусу та збереження оцінки
    movie.status = "WATCHED"
    movie.userRating = rating
    movie.dateWatched = getCurrentDate()

    // 4. Збереження змін у базі
    database.save(movie)

    return "Успішно: Фільм додано до переглянутих"
```

## 5. Тестування

1.  **Тест додавання:** Користувач вводить назву "Матриця" і натискає "Зберегти" → Фільм з'являється у списку "Планую".
2.  **Тест валідації рейтингу:** Користувач вводить оцінку "12" → Система видає помилку і не зберігає результат.
3.  **Тест дублікатів:** Користувач намагається додати фільм, який вже є у списку → Система пропонує відкрити існуючий запис замість створення нового.

## 6. Висновки

Для розробки цього трекера найкраще підходить модель **Agile**.
Вона дозволяє випустити базову версію (MVP) лише зі списками та пошуком, а в наступних ітераціях додавати складні функції (статистику, рекомендації, інтеграцію з API кінотеатрів). Waterfall не підходить, оскільки вимоги до інтерфейсу та функціоналу розважальних додатків часто змінюються під час розробки на основі відгуків користувачів.
